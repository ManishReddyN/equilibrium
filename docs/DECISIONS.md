# Decisions Log

One line per deviation from `roommate-app-execution-plan.md`, with rationale. Newest entries at the bottom.

- **2026-07-18** — Execution environment is Windows 11 (no macOS/Xcode). iOS native builds, iOS Simulator runs, and the Fastlane iOS lane (`app/ios/fastlane/Fastfile`) cannot be executed on this machine under any configuration, including Docker: Apple's EULA prohibits virtualizing macOS on non-Apple hardware, and there is no Linux/Windows build of Xcode. Phase 0-4 iOS verification gates (`run-ios`, simulator checks) will be documented as blocked rather than run. iOS work is limited to writing source/config that a macOS machine could later build.
- **2026-07-18** — Ruby and JDK are not installed on this machine at plan start, so the Android toolchain and Fastlane (Ruby-based) cannot run yet either. Android-side verification gates are blocked until those are installed (a Docker image with JDK + Android SDK is a viable path for Android specifically, revisit at Phase 1/6).
- **2026-07-18** — Reusing the existing Supabase project from the prior `roomie` project (ref `zubxuqshcyniosmdztmw`) instead of provisioning a new Supabase project for Phase 2, per user direction. `roomie`'s existing schema (`households`, `roommates`, `chores`, `chore_log`) differs substantially from this plan's schema (`households`, `profiles`, `chores`, `assignments`, `audit_ledger`, `feedback_queue`, `market_listings`, `notification_outbox`). The exact merge/replace strategy for the live schema is deferred to Phase 2 and will be logged here when decided.
- **2026-07-18** — Executing phase-by-phase with a stop-and-report checkpoint after each Verification Gate, per user preference, rather than pushing through all phases autonomously.
- **2026-07-18** — Added `__tests__` to `app/tsconfig.json`'s `include` array (plan section 0.4 lists only `["src", "index.js", "*.config.js"]`). Without it, `@typescript-eslint/recommended-type-checked` (added in 0.5) cannot type-check `__tests__/App.test.tsx`, since Jest test files are program entry points that nothing else imports, so TS never pulls them into the type-checked program otherwise.
- **2026-07-18** — Replaced the RN CLI's default boilerplate `App.tsx` (which imports the untyped `react-native/Libraries/NewAppScreen` module) with a minimal typed placeholder, to satisfy strict typed-linting without suppression comments. It will be replaced again by the composed app shell in Phase 1/3 regardless.
- **2026-07-18** — Pinned all RN-template-generated `devDependencies` in `app/package.json` to their exact resolved lockfile versions (removed `^`/`~` ranges), per the plan's hard "pin every dependency" rule (section 0, item 4). The plan's Phase 1 install list only pins newly-added packages explicitly; this extends the same rule to the CLI-generated baseline.
- **2026-07-18** — Installed `babel-plugin-module-resolver@5.0.2` in Phase 0 (rather than its listed spot in Phase 1's batch install), since Phase 0 section 0.4 explicitly requires configuring it for path aliases immediately. Phase 1 will find it already present at the correct pinned version.
